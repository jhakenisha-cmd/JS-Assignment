Approach and Complexity AnalysisQuestion 1: Digit GatekeeperApproach: I implemented a for loop to iterate through the range $[L, R)$. For each number divisible by $K$ , a while loop extracts digits to calculate their sum and check for the presence of the digit $0$. A nested for loop then verifies if the digit sum is prime.Complexity: $O(N \cdot D + \text{PrimeCheck})$, where $N$ is $(R - L)$ and $D$ is the number of digits (max 7). Since the maximum digit sum is small (54), the prime check is effectively constant time, $O(1)$.Question 2: Roll-Seed LockApproach: The logic uses a for loop to repeat a transformation exactly 3 times. Inside, an if-else block checks for parity (even/odd) and applies the specified formula. Finally, I extracted the middle digit using Math.floor(current / 10) % 10 to compare it with the seed.Complexity: $O(1)$, because the loop always runs exactly 3 times regardless of the input size.Question 3: Mirror CorridorApproach: I used a for loop to test values of $X$ from 0 to 10000. For each $X$, I checked if $(N+X)$ is divisible by $K$. To check for a palindrome without using string methods, I used a while loop to reverse the digits of $(N+X)$ and compared it to the original.Complexity: $O(X_{max} \cdot D)$, where $X_{max}$ is 10,000 and $D$ is the number of digits in $(N+X)$.Question 4: Fare CalculatorApproach: This follows a sequential set of conditional rules (if statements) to modify the base fare. To round up to the nearest multiple of 5, I used a while loop that increments the fare until fare % 5 === 0.Complexity: $O(1)$, as the number of operations is fixed and the rounding loop runs a maximum of 4 times.Question 5: Skipping NumbersApproach: I used a while loop that continues until the cumulative sum reaches $N$. Inside, a counter m increments, and an if statement checks if m is divisible by (seed + 2) to decide whether to add it to the sum.Complexity: $O(\sqrt{N})$, as the sum of the first $m$ integers grows quadratically ($m^2/2 \approx N$).Question 6: Contest Score JudgeApproach: The score is calculated using the linear formula $3a + b - 2c$. Simple if conditions handle the negative score floor , the total attempts penalty , and the final PASS/FAIL status determination.Complexity: $O(1)$, as it involves only basic arithmetic and a few comparisons.
